# 贡献指南

感谢你愿意为这个产品创意仓库贡献力量！我们欢迎以下类型的贡献：

- 添加新的 AI 产品想法
- 完善现有想法的描述（如补充技术可行性、风险评估）
- 修正错误（如编号重复、标签错误）
- 提供相关竞品信息或参考文献

## 如何提交新创意

1. **先搜索**：在 `ideas.md` 中搜索关键词，确保该创意未被收录。
2. **开 Issue**：在 [Issues](../../issues) 页面点击 “New issue”，选择模板 `new_idea.md` 并填写。
3. **讨论**：维护者和其他贡献者可能会在 Issue 下讨论，若获得至少 2 个 👍 或维护者批准，即可进入下一步。
4. **提交 Pull Request**：
   - Fork 本仓库
   - 在 `ideas.md` 末尾添加新行（按编号递增）
   - 提交 PR，标题格式：`Add idea: [产品名称]`
   - 在 PR 描述中链接对应的 Issue

## 完善现有想法

如果你发现某个想法的“技术可行性”或“状态”需要更新，或者你想补充详细描述，请直接开 Pull Request，并在 PR 中说明理由。

## 格式规范

- 编号：自动递增，从当前最大编号 +1
- 产品名称：简洁明了，不超过 20 字
- 一句话定位：不超过 30 字
- 目标场景、目标用户：使用逗号分隔多个值
- 技术可行性：仅限 “高”、“中”、“低”
- 状态：仅限 “✅ 可尝试”、“⚠️ 风险过高暂缓”、“❌ 已弃用”、“🔬 待评估”

## 行为准则

请遵守 [行为准则](./CODE_OF_CONDUCT.md)。

感谢你的贡献！
